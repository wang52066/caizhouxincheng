## 📋 准备工作(1 分钟)

打开浏览器,访问: **https://github.com**

- 如果没有 → 点「Sign up」注册(用邮箱,免费)

---

## 🎯 部署步骤(2 分钟)

### 第 1 步:创建新仓库

1. 登录后,点右上角 **「+」→「New repository」**
2. 填写:
   - **Repository name**(仓库名):`caizhouxincheng`(必须填这个,后面链接要用)
   - **Description**:选填,可不填
   - **Public**(必须选这个!)
   - **不要**勾选「Add a README file」
3. 点 **「Create repository」**

> ⚠️ 仓库名一定要是 `caizhouxincheng`,因为这样你的网址会是:
> `https://你的用户名.github.io/caizhouxincheng/`

---

### 第 2 步:上传文件

进入新创建的仓库页面后:

1. 找到 **「uploading an existing file」** 链接(在中间位置,蓝色小字),点它
   > 或者点 **「Add file」→「Upload files」**

2. 打开文件夹 `caizhouxincheng-website`(我刚给你放到了桌面)

3. **把里面的所有内容(4 个 html + css 文件夹 + js 文件夹)全部拖进浏览器窗口**

> 💡 关键:拖的是文件夹的**内容**,不是把整个文件夹拖进去
> 拖完后,浏览器里应该看到:
> ```
> about.html
> contact.html
> css/
> index.html
> js/
> services.html
> ```

4. 滚动到底部,找到 **「Commit changes」** 按钮,**直接点**(不用改任何东西)

5. 等 10-30 秒,文件上传完成,会自动跳回仓库主页

---

### 第 3 步:开启 Pages 服务

1. 在仓库页面,点顶部 **「Settings」**(注意:不是 Account Settings,是仓库的 Settings)

2. 左侧菜单找到 **「Pages」**(在「Code and automation」分类下)

3. 在「Branch」那里:
   - 第一个下拉框选 **「main」**
   - 第二个下拉框保持 **「/ (root)」**
   - 点 **「Save」**

4. 等 1-2 分钟,页面会刷新,出现一个绿色提示框:
   ```
   ✅ Your site is live at https://你的用户名.github.io/caizhouxincheng/
   ```

**这就是你的永久网址!** 🎉

## ❓ 遇到问题?

### 找不到 Pages 菜单?
- Settings 在最上面一排,不是右上角的头像
- 在「Code and automation」分类下找「Pages」

### 上传后看不到文件?
- 刷新一下浏览器(`Ctrl+R` 或 `F5`)

### 链接打开是 404?
- 仓库名必须是 `caizhouxincheng`,不能是别的
- 等待 2-3 分钟,GitHub 在编译

### 样式显示异常?
- 等 1 分钟再刷新
- 检查 css 和 js 文件夹是不是和 html 在同一层
